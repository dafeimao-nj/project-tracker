# 任务：关闭 Docker 沙箱模式

## 目标
解决 "Sandbox mode requires Docker, but docker command not found" 错误，让 Agent 能正常启动运行。

## 关键操作
1. **诊断**：确认配置文件中未显式设置 sandbox，系统默认启用 Docker sandbox
2. **尝试 config.patch**：路径被保护（protected），无法通过 API 修改
3. **直接编辑 openclaw.json**：添加 `agents.defaults.sandbox.mode = "off"`
4. **Gateway 重启**：SIGUSR1 热重载生效

## 结论
✅ Docker 沙箱已关闭，Agent 可在无 Docker 环境下正常运行。

## 重要发现
1. **SIGUSR1 热重载对 sandbox 配置不生效！** 必须用 `openclaw gateway restart`（完整重启 LaunchAgent）
2. **系统存在两个配置文件！** `~/.openclaw/openclaw.json` 优先级高于 `~/.qclaw/openclaw.json`
   - `~/.openclaw/openclaw.json` 里原来设置的是 `sandbox.mode = "all"`（强制Docker）
   - 一直改错文件了！改了 `~/.qclaw/` 的但实际读的是 `~/.openclaw/` 的
3. 修复 `~/.openclaw/openclaw.json` 的 sandbox.mode=off 后问题解决
