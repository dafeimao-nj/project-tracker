# 任务：OpenClaw 安全配置修复

## 目标
修复飞书群安全漏洞（open 模式 + elevated 工具）和控制台不安全的 auth 配置。

## 发现的问题

### 问题 1：飞书群策略过于开放
- 配置文件：`~/.openclaw/openclaw.json`（注意不是 ~/.qclaw/ 那个）
- `channels.feishu.groupPolicy = "open"` — 任何人都能在群里触发 Agent
- 无 `groupAllowFrom` / `groupSenderAllowFrom` — 没有白名单限制
- DM 策略已经是 allowlist（正确），但群策略是 open（危险）

### 问题 2：Control UI allowedOrigins 含 "null"
- `gateway.controlUi.allowedOrigins = ["null", "file://"]`
- "null" origin 是安全隐患，恶意网页可利用

## 执行的修复

### 修复 1：飞书群策略收紧
```json
"groupPolicy": "allowlist"                          // 原: "open"
"groupAllowFrom": ["ou_40a1e80ba14d32c6cd7d9a3bd8989bba"]
"groupSenderAllowFrom": ["ou_40a1e80ba14d32c6cd7d9a3bd8989bba"]
```

### 修复 2：移除 null origin
```json
"allowedOrigins": ["file://"]                       // 原: ["null", "file://"]
```

## 结果
✅ Gateway 完全重启成功，零错误零警告
✅ 飞书群现在只有用户本人能触发 Agent
✅ Control UI 不再接受 null origin

## 重要经验
- 系统有两个配置文件：`~/.openclaw/openclaw.json`（高优先级）和 `~/.qclaw/openclaw.json`
- 飞书配置在 `~/.openclaw/` 中，之前检查 `~/.qclaw/` 时没看到 feishu 配置
- SIGUSR1 热重载对某些配置项不生效，必须用 `openclaw gateway restart` 完全重启
