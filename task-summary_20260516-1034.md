# 微信推送问题排查与修复 - 2026-05-16

## 目标
修复定时任务微信推送不生效的问题，实现飞书+微信双渠道自动推送。

## 问题排查过程（08:00~10:34）

### 第一轮：渠道名错误
- 初始配置：微信任务用了 `wechat-access`（服务号bot）
- 实际应该用：`openclaw-weixin`（个人微信ClawBot AI）

### 第二轮：PDF指令丢失
- 修改渠道时把PDF生成指令从message里丢了
- 教训：修改任务时必须完整保留原有格式指令

### 第三轮：跨渠道限制发现
- 发现OpenClaw安全限制：从飞书session无法用message工具跨渠道发微信
- 方案：拆成独立cron任务，各走自己的channel + announce自动投递

### 第四轮：改错渠道
- 把微信任务从 `openclaw-weixin` 改成了 `wechat-access`
- wechat-access 虽然有token/wsUrl，但announce投递路径不通
- 系统报 delivered=true 但实际收不到（假成功）

### 第五轮：最终修复 ✅
- 改回 `openclaw-weixin` + announce纯文本投递 + message工具发PDF
- 截图验证：纯文本摘要和PDF文件都成功收到

## 关键经验教训
1. **openclaw-weixin 是唯一可用的微信渠道**，不要改成 wechat-access
2. **announce模式可以跨渠道投递纯文本**，message工具从飞书session发微信也能成功（在openclaw-weixin上）
3. **delivered=true 不代表真正送达**，必须用户确认才算
4. **修改任务要做端到端验证**，不能只看系统返回值
5. **一次改到位**，反复修补会让用户失去信任
