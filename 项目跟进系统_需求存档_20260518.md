## 一、项目背景

**用户身份**：数科公司整体业务运营负责人，负责 AI 能力落地的推进工作

**核心场景**：需要跟进非常多的工作事项，每个事项都需要有专门人跟进，以及项目的最新动态和进展、预计完成时间等

**目标**：创建一个工作事项管理系统，梳理提示词让编程智能体完成项目编写

## 二、已确认需求

### 1. 核心字段

**基础字段**（必须有）：
1. **事项名称** - 文本
2. **事项类型** - 单选（风控模型的搭建 / 智能客服 / 智能巡检 / 可扩展）
3. **负责人** - 人员选择
4. **状态** - 单选（未启动 / 进行中 / 遇阻 / 待验收 / 已完成 / 已取消）
5. **优先级** - 单选（低 / 中 / 高 / 紧急）
6. **创建时间** - 日期（自动）
7. **预计完成时间** - 日期
8. **实际完成时间** - 日期

**扩展字段**（待确认）：
- 最新进展 - 多行文本（保留历史记录）
- 关联项目 - 是否需要？
- 协作人 - 是否有多人协作？
- 进度百分比 - 是否需要量化？

### 2. 功能需求

**视图**：
- ✅ 看板视图（按状态分列展示）

**跟进记录**：
- ✅ 保留历史记录
- ✅ 需要「跟进记录」子表

**提醒功能**：
- ✅ 超期提醒（超过预计完成时间未完成）

**统计看板**：
- ✅ 总事项数
- ✅ 各状态数量
- ✅ 完成率

**现有数据**：
- ❌ 无（从零开始）

## 三、技术方案

### 已选方案：本地 Web 应用

**理由**：
- 飞书多维表格权限问题太多（今天踩了一下午坑）
- 本地 Web 应用完全可控，无权限问题

**技术选型**：
- **前端**：React + Tailwind CSS（看板视图 + 响应式设计）
- **后端**：Python FastAPI（轻量级，开发快）
- **数据库**：SQLite（本地文件，无需安装）
- **部署**：本地运行，一键启动
